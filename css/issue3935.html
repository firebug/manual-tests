Issue 3935: Strange behaviour when disabling CSS rules

Steps to reproduce

  1. Inspect the blue <div> above (#element1)
  2. Switch to the HTML panel and there to the Style side panel (if you're not already there)
  3. Disable the background property
  4. Create a new background property by double-clicking into the rule's body and give it the value #8CFF8C
  5. Leave editing by clicking somewhere else
    ⇒ The background property will be added, but the disabled one is kept
  6. Enable the old background property again

Observed Result

  • The new background property is overwritten by the old one

Expected Result

  • The new property is added and enabled (already working)
  • After step 5 the old property should disappear
