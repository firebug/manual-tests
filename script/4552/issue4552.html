Issue 4552: Error breakpoints should not hit unless there is an error

Steps to reproduce:

  1. Open Firebug
  2. Enable and switch to the Console panel
  3. Click the Error button above to rise an error
  4. Inside the Console panel click on the dot besides the error to create an error breakpoint
  5. Click the Error button again
  6. Click the Continue button inside the Script panel
  7. Click the No error button above to properly call the same code/function

Observed results

  • Error doesn't happen, but debugger still breaks

Expected results

  • After step 5: Script execution halts on the error breakpoint and Firebug switches to the Script panel
  • After step 7: Debugger shouldn't break
